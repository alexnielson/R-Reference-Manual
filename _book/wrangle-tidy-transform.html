R Reference Manual

Wrangle: Tidy & Transform

“Tidying your data means storing it in a consistent form that matches the semantics of the dataset with the way it is stored. In brief, when your data is tidy, each column is a variable and each row is an observation. Tidying data is important because the consistent structure lets you focus your struggle on questions about the data, not fighting to get the data into the right form for different functions.” -Garrett Grolemund & Hadley Wickham, R for Data Science

“Transformation includes narrowing in on observations of interest (like all people in one city, or all data from the last year), creating new variables that are functions of existing variables (like computing velocity from speed and time), and calculating a set of summary statistics (like counts or means).” -Ibid.

Tidy

Class, Mode, and Type

  • base::as.*(): Coerces an object to the indicated type.
    • base::as.*() family functions: as.array(), as.data.frame(), as.Date(), as.factor(), as.list(), as.matrix(), as.POSIX(), as.POSIXct().
    • Note: Consider tibble::as_tibble() preferable to base::as.data.frame().
  • base::class()/unclass(): View, assign, or remove an object class.
  • methods::as(): Force an object to belong to a class.
  • tibble::as_tibble(): Coerce lists and matrices to data frames.
    • Note: Preferable to base::as.data.frame().

Manipulate Data Format

  • dplyr::arrange(): Arrange rows by variable, in ascending order.
  • purrr::keep(): Keep or discard elements using a predicate function.
  • purrr::transpose(): Turn a list-of-lists inside-out.
  • splitstackshape::cSplit(): Split concatencated values into separate values.
  • stats::na.omit(): Remove rows with NA values.
  • tidyr::gather(): Gather columns.
  • tidyr::separate(): Separate one column into multiple columns.
  • tidyr::spread(): Spread across multiple columns.
  • tidyr::unite(): Unite multiple columns into one.

Transform

Summarize (incl. Math & Statistics)

  • base::abs(): Compute absolute value.
  • base::colMeans(): Compute the column mean.
  • base::colSums(): Compute the column sum.
  • base Comparison Operators: Binary operators which allow the comparison of values in atomic vectors.
    • <, >, <=, >=, ==, !=.
    • Note that identical() and all.equal() should be used in tests (such as if expressions) where a single TRUE or FALSE is required, rather than == and !=.
  • base::max(): Return the maximum value.
  • base::mean(): Compute the mean.
  • base::min(): Return the minimum value.
  • base::round(): Round numbers.
  • base::rowMeans(): Compute the row mean.
  • base::rowSums(): Compute the row sum.
  • base::sqrt(): Compute square root.
  • base::sum(): Sum elements.
  • dplyr::n(): Get the number of observations in a current group (must be used within summarise(), mutate(), or filter()).
  • dplyr::n_distinct(): Count the number of unique values in a vector.
  • stats::aggregate(): Compute summary statistics of data subsets.
  • stats::cor(): Correlation.
  • stats::cov(): Covariance.
  • stats::rnorm(): Generate a random normal distribution.
  • stats::var(): Variance.

Merging, Set Operations, & Adding to Existing Data

  • base::append(): Add elements to a vector.
  • base::cbind(): Combine objects by column.
  • base::intersect(): Combine data shared in common between two datasets.
    • Note: Similar to dplyr::semi_join().
  • base::merge(): Merge two data frames.
    • Note: dplyr::join functions are an alternative to merge().
  • base::rbind(): Combine objects by row.
  • base::setdiff(): Find the difference between two vectors.
    • Note: Similar to dplyr::anti_join().
  • base::setequal(): Check two vectors for equality.
  • base::union(): Combine two datasets without duplicating values.
  • dplyr::bind(): Bind multiple data frames by row and column.
    • dplyr::bind() family functions: bind_rows(), bind_cols(), and combine().
  • tibble::add_column(): Add columns to a data frame.
  • tibble::add_row(): Add rows to a data frame.

Tidy or Transform

Dates and Datetimes

  • anytime::anytime(): Parse POSIXct or Date objects from input data.
  • base::as.Date(): Date conversion to and from character.
  • base::as.POSIX(): Date-time conversion.
  • base::as.POSIXct(): Setting default for UTC and 1970.
  • base::date(): Get the current system date and time.
  • base::diff(): Compute the difference between two objects (useful with dates).
  • base::months(): Extract the month names.
  • base::quarters(): Extract the calendar quarters.
  • base::strptime(): Date-time conversion to and from character.
  • base::Sys.Date(): Get the current date in the current time zone.
  • base::Sys.time(): Get the absolute date-time value (which can be converted to various time zones and may return different days).
  • base::weekdays(): Extract weekday names.
  • lubridate::date(): Get/set the date component of a date-time.
  • lubridate::day(): Get or set the day component of a datetime.
  • lubridate::parse_date_time(): User friendly date-time parsing functions that can accomodate parsing multiple dates in different formats.
    • lubridate::fast_strptime(): Fast C parser of numeric formats only that accepts explicit format arguments, just as base::strptime().
    • lubridate::make_date(): Create dates from numeric representations.
    • lubridate::make_datetime(): Create date-times from numeric representations.
    • lubridate::parse_date_time2(): Fast C parser of numeric orders.
  • lubridate::month(): Get or set the month component of a datetime.
  • lubridate::quarter(): Get or set the fiscal quarter or semester component of a datetime.
  • lubridate::year(): Get or set the year component of a datetime.
  • lubridate::ymd(): Parse dates with year, month, and day components. + Related formats: ydm(), mdy(), myd(), dmy(), dym(), yq().
  • lubridate::ymd_hms(): Parse date-times with year, month, dat, hour, minute, and second components. + Related formats: ymd_hm(), ymd_h(), dmy_hms(), dmy_hm(), dmy_h(), mdy_hms(), mdy_hm(), mdy_h(), ydm_hms(), ydm_hm(), ydm_h().

Selecting & Subsetting

  • base::subset(): Subset vectors, matrices, and data frames.
  • dplyr::first(): Extract the first element of a vector.
  • dplyr::last(): Extract the last element of a vector.
  • dplyr::nth(): Extract the nth element of a vector.

Strings

  • base::cat(): Concatenate and print.
  • base::chartr(): Change certain characters.
  • base::grep(): Pattern matching and replacement.
    • grep() family functions: grepl(), sub(), gsub(), regexpr(), gregexpr(), and regexec().
  • base::tolower(): Convert to lowercase.
    • Note: stringr::str_to_lower() is an alternative.
  • base::toupper(): Convert to uppercase.
    • Note: stringr::str_to_upper() is an alternative.
  • stringr::str_detect(): Detect the presence or absence of a pattern in a string.
    • Note: Control the pattern argument options with regex() (i.e., str_detect(x, regex(pattern, ignore_case = TRUE)).
  • stringr::str_to_lower(): Convert to lower case.
  • stringr::str_to_title(): Capitalize the first letter.
  • stringr::str_to_upper(): Convert to upper case.

Test

  • base::all(): Are all values true?
  • base::any(): Are any values true?
    • Note: Use any(is.na(data.frame)) to determine if there are any NA values in a data frame.
  • base::setequal(): Check two vectors for equality.
  • purrr::every(): Do every or some elements of a list satisfy a predicate?
  • stats::complete.cases(): Find complete cases (rows without NA values).

View

  • tibble::glimpse(): Get a glimpse of the data.
    • Note: Similar to utils::str().
  • utils::head(): View the first observations in a data frame.
  • utils::str(): Display the structure of an R object.
    • Note: Similar to tibble::glimpse().
  • utils::tail(): View the last observations in a data frame.

Packages

  • anytime: Date converter.
  • base: Base R functions.
  • chron: Chronological objects which can handle dates and times.
  • dplyr: Data manipulation.
  • forcats: Tools for working with categorical variables.
  • lubridate: Functions to work with date-times and time-spans.
    • lubridate uses character formatting similar to strptime(), though there are some differences. To see lubridate’s formatting, type ?parse_date_time into the R Console.
  • methods: Formal methods and classes.
  • purrr: Functional programming tools.
  • splitstackshape: Stack and reshape datasets after splitting concatenated values.
  • stats: Statistical functions.
  • stringr: Working with strings (see also stringr.tidyverse.org).
  • tibble: Simple data frames with stricter checking and better formatting than the traditional data frame.
  • tidyr: Tidy data.
  • utils: Various programming utilities.
  • xts: Provide for uniform handling of R’s different time-based data classes by extending zoo.
  • zoo: For regular and irregular time series.

  • dplyr::count(): Count/tally observations by group.
  • dplyr::tally(): An alternative to count().
  • dplyr::distinct(): Select distinct rows.
  • dplyr::filter(): Return rows with matching conditions.
  • dplyr::group_by(): Group by one or more variables.
  • dplyr::Join Functions: Join two tables.
    • Filtering Joins:
      • anti_join(): Return all rows from x where there are not matching values in y, keeping just columns from x.
      • semi_join(): Return all rows from x where there are matching values in y, keeping just columns from x. A semi join differs from an inner join because an inner join will return one row of x for each matching row of y, where a semi join will never duplicate rows of x.
    • Mutating Joins:
      • full_join(): Return all rows and all columns from both x and y. Where there are not matching values, returns NA for the one missing.
      • inner_join(): Return all rows from x where there are matching values in y, and all columns from x and y. If there are multiple matches between x and y, all combination of the matches are returned.
      • left_join(): Return all rows from x, and all columns from x and y. Rows in x with no match in y will have NA values in the new columns. If there are multiple matches between x and y, all combinations of the matches are returned.
      • right_join(): Return all rows from y, and all columns from x and y. Rows in x with no match in y will have NA values in the new columns. If there are multiple matches between y and x, all combinations of the matches are returned.
  • dplyr::mutate(): Add new variables.
  • dplyr::rename(): Rename variables by name (a modification of select()).
  • dplyr::select(): Select/rename variables. Helper functions include: starts_with(), ends_with(), contains(), matches(), num_range(), and one_of().
  • dplyr::summarise(): Reduce multiple values to a single value.
    • Useful with aggregate functions.
